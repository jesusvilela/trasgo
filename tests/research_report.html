Key Findings

1. Protocol FILTER is the hardest opcode.
T4 is consistently the weakest test across ALL models. Even §1-ADVANCED models score 1/3–1/4. FILTER requires parsing a §P instruction, extracting and ranking a numeric field across packets, and applying top-k selection — the closest §1 gets to "computation" vs "comprehension." Only frontier models (GPT-4o, Claude Opus) execute §P reliably.
2. State tracking is robust across all scales.
T9 (triple-delta state consistency) passes at 4B, 27B, and frontier alike. Even models that fail grammar induction can track orbit/fuel/power/payload through 3 sequential deltas. JSON comprehension and state accumulation are separate capabilities from §1 grammar induction.
3. The scale threshold is NOT parameter count alone.
rnj-1-instruct (unknown params) scores 83.3% = §1-ADVANCED. ig-bundlellmv2-hamiltonians scores 66.7% with perfect state machine scores but fails calibration. Architecture, training data, and RLHF alignment interact with parameter count. The threshold is better understood as a capability requirement: sufficient in-context learning capacity to induce a novel grammar from 3 examples.
4. Cross-provider portability confirmed.
Same boot seed, same results across: LM Studio (local GGUF), DeepSeek API, OpenAI API, Anthropic API, DeepSeek Android app. The §1 ISA is architecture-portable — the boot seed compiles equivalently on different "CPUs" with different "microarchitectures."
