# Factory Copilot

Purpose: show Trasgo as an operator-facing predictive maintenance shell, not just a codec.

Command surface:

```bash
trasgo demo run factory-copilot
trasgo "run the factory copilot demo"
```

What it demonstrates:

- `ctx_context` confirmation: the same operational state is measured in narrative tokens versus codec tokens.
- Compression proof: a smaller §1 state packet still preserves the intervention decision.
- Functional gain: the operator gets one ranked action brief instead of stitching together dashboards by hand.
- Translating line telemetry and inventory drift into a compact §1 packet.
- Turning that packet into an action plan that operations can execute the same shift.
- Quantifying avoided downtime and intervention cost so the recommendation is economically legible.

Why it matters:

- Operative value: schedules a controlled micro-stop before a bearing event becomes a line outage.
- Economic value: makes the maintenance decision defensible with same-day avoided-loss numbers.
